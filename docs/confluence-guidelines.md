# Confluence 운영 규칙

## 기본 원칙

### Confluence = 단일 진실 공급원 (Single Source of Truth)
- 모든 공식 문서는 Confluence에 작성
- 다른 도구(Google Drive, Notion, Slack)의 중요 정보는 Confluence로 이관
- 문서를 찾을 때는 Confluence를 먼저 검색

### 문서화 우선 문화
- 구두 전달 → Confluence 문서화
- 중요한 Slack 스레드 → Confluence 페이지로 변환
- 개인 노트 → 팀 공유 문서로 승격

## 문서화 대상

### 필수 문서화 항목
- 프로젝트 개요 및 목적
- 기술 스펙 및 아키텍처
- API 문서
- 배포 프로세스
- 트러블슈팅 가이드

### 선택적 문서화 항목
- 회의록
- 의사결정 기록
- 리서치 결과
- 학습 자료

## 페이지 구조

### 공간(Space) 구조
```
POPUP STUDIO
├── 프로젝트 A
│   ├── 개요
│   ├── 기술 문서
│   ├── 회의록
│   └── 주간 보고서
├── 프로젝트 B
└── 공통
    ├── 개발 가이드
    ├── 운영 규칙
    └── 온보딩
```

### 페이지 명명 규칙
- 명확하고 구체적인 제목
- 날짜 포함 시: YYYY-MM-DD 형식
- 예: `2025-11-05 주간 보고서`

## 작성 가이드

### 문서 템플릿 활용
- Confluence 기본 템플릿 사용
- 팀 공통 템플릿 적용

### 내용 작성 원칙
- 간결하고 명확하게
- 코드 예제 포함
- 스크린샷 활용

### 접근 권한 관리
- 프로젝트별 권한 설정
- 민감 정보 별도 관리

## 정보 중앙화 전략

### 산재된 정보 마이그레이션

#### Google Drive → Confluence
**대상**:
- 프로젝트 관련 문서
- 기술 문서
- 회의록
- 공유가 필요한 스프레드시트

**마이그레이션 절차**:
1. Google Drive에서 중요 문서 식별
2. Confluence에 적절한 Space/페이지 생성
3. 문서 내용 복사 및 포맷 정리
4. 원본 Google Drive 문서에 "이 문서는 Confluence로 이관되었습니다" 노트 추가
5. Confluence 링크 명시

**AI Agent 활용**:
```
Claude Code에게 요청:
"Google Drive의 [문서 링크]를 Confluence [Space]로 마이그레이션해줘"
```

#### Notion → Confluence
**대상**:
- 프로젝트 노트
- 팀 위키
- 프로세스 문서

**마이그레이션 절차**:
1. Notion에서 Markdown으로 내보내기
2. Confluence에 페이지 생성 후 가져오기
3. 이미지, 테이블 재정리
4. 원본 Notion 페이지에 마이그레이션 완료 표시

#### Slack → Confluence
**대상**:
- 중요한 의사결정 스레드
- 기술 토론 내용
- 문제 해결 과정

**변환 절차**:
1. Slack 스레드 복사
2. Confluence에 페이지 생성
3. 컨텍스트와 함께 정리된 형태로 문서화
4. 원본 Slack 스레드에 Confluence 링크 공유

**AI Agent 활용**:
```
Slack 스레드를 복사한 후:
"이 Slack 대화를 Confluence 페이지로 정리해줘"
```

#### 로컬 문서 → Confluence
**대상**:
- 개인 PC의 기술 문서
- 로컬 README 파일
- 개인 메모 중 공유 가치가 있는 내용

**공유 절차**:
1. 공유 가치 판단
2. Confluence에 업로드
3. 적절한 카테고리 분류
4. 팀에 공유

### 정보 발견 및 이관 프로세스

#### 1. 주간 정보 점검
매주 팀별로 다음 확인:
- 이번 주 Google Drive에 생성된 중요 문서
- Slack에서 문서화가 필요한 스레드
- 개인이 작성한 공유 가치 있는 문서

#### 2. AI Agent 지원
Claude Code를 활용한 자동화:
- 산재된 정보 식별
- 마이그레이션 템플릿 제공
- 문서 구조화 지원

#### 3. 분기별 대청소
3개월마다:
- 오래된 Google Drive 문서 검토
- 미사용 Notion 페이지 정리
- Confluence로 통합 가능한 문서 일괄 이관

### 마이그레이션 우선순위

**높음 (즉시 이관)**:
- 프로젝트 핵심 문서
- 자주 참조되는 기술 문서
- 온보딩 관련 자료

**중간 (계획적 이관)**:
- 과거 프로젝트 회의록
- 리서치 자료
- 학습 자료

**낮음 (선택적 이관)**:
- 개인 메모
- 일회성 문서
- 임시 자료

## 문서화 자동화

### AI Agent 활용 예시

#### Slack 스레드 자동 문서화
```bash
# Slack에서 중요 스레드 발견 시
/save-slack-thread [스레드 링크]

# Claude Code가:
# 1. Slack API로 스레드 내용 가져오기
# 2. 문서 구조화
# 3. Confluence 페이지 생성
# 4. Jira 이슈와 연결 (해당되는 경우)
```

#### 주간 보고서 자동 아카이빙
```bash
/weekly-report

# Claude Code가:
# 1. Jira에서 완료 이슈 조회
# 2. 보고서 생성
# 3. Confluence에 자동 저장
# 4. 팀 Space에 링크 공유
```

### 문서 연결 관리

#### Cross-referencing
- Jira 이슈 ↔ Confluence 문서 양방향 링크
- 관련 문서끼리 링크로 연결
- 시리즈 문서는 부모-자식 관계 설정

#### 태그 및 라벨 활용
- 프로젝트명 태그
- 문서 타입 라벨 (기술문서, 회의록, 가이드 등)
- 우선순위 라벨 (필독, 참고, 보관 등)
