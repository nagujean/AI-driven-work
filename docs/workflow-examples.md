# 업무 플로우 예시

## 일일 업무 루틴

### 출근 후 업무 시작
```bash
# 1. Claude Code 실행
claude

# 2. 일일 스탠드업 확인
/daily-standup

# 3. 미할당 이슈 중 선택하여 할당
/assign-me PROJ-123

# 4. 업무 시작
```

### 업무 진행 중
- Jira 이슈 상태 업데이트
- 진행 상황을 코멘트로 기록
- 필요시 Sub-Task 생성

### 업무 종료 전
- 진행 중인 이슈에 오늘 작업 내용 코멘트
- 완료한 이슈는 Done으로 상태 변경
- 내일 할 작업 메모

- 질문 및 피드백

## 이슈 생성부터 완료까지

### 1. 이슈 발견 및 생성
```
상황: 버그 발견 또는 개선 아이디어

1. Jira에서 새 이슈 생성
   - 제목: 명확하고 구체적으로
   - 설명: 재현 방법 또는 요구사항
   - 타입: Bug, Task, Story 중 선택
   - 우선순위 설정

2. 담당자: 미할당 상태로 두기
```

### 2. 이슈 할당
```bash
# 다른 팀원이 /daily-standup으로 확인
/daily-standup

# 처리 가능한 이슈 선택
/assign-me PROJ-456
```

### 3. 이슈 처리
```
1. 이슈 상태를 "In Progress"로 변경
2. 작업 수행
3. 진행 상황 코멘트로 공유
4. 코드 커밋 시 이슈 키 포함
   예: git commit -m "PROJ-456: 버그 수정"
```

### 4. 이슈 완료
```
1. 작업 완료 확인
2. 테스트 완료
3. PR 생성 및 리뷰 요청
4. 머지 후 이슈 상태 "Done"으로 변경
5. 완료 코멘트 작성
```

## 긴급 상황 대응

### 프로덕션 버그 발생
```
1. 긴급 이슈 생성
   - 우선순위: Highest
   - 타입: Bug
   - 라벨: production, urgent

2. Slack으로 팀에 알림
3. 즉시 담당자 할당
4. 핫픽스 진행
5. 사후 분석 문서 Confluence에 작성
```

### 서버 장애
```
1. 모니터링 도구 확인
2. Jira 인시던트 티켓 생성
3. 장애 대응 진행
4. 복구 후 원인 분석
5. 재발 방지 계획 수립
```
